Mounts

Add a mount's .spr — works for both halter mounts (Lion, Cerberus…) and intrinsic mounts (Dragon, Peco, Gryphon…). Use the original Gravity sprite; re-indexed "corrected" sprites from custom packs change the baked-in layout and won't line up. The rider is detected by index alignment — slots whose baked colour matches the base sprite follow the character's recolour exactly, so the rider stays identical to the unmounted form. Everything else is the mount; theme it with your scheme and lock any areas you want to keep.

Then in Schemes → Bake all, tick Also create mount palettes to export the character and every mount together (one .zip, each palette named for its own sprite + gender + #).

Scheme library (0 saved)

A scheme stores a colour theme (per-area hues), not exact slots — so it applies to any class. Save the current look, or roll a batch, then click a scheme to apply it to the loaded sprite. After tweaking the palette, Save current adds a new scheme, or click the ✎ on a scheme to overwrite that one in place.

Bake all for current class

Applies every saved scheme to the loaded class and writes base + gender + _# palettes (uses the Export tab's structure + base name). e.g. …_0.pal … _699.pal · With mounts on, each scheme also writes a palette for every sprite added in the Mount tab, keeping clothing identical to the character.

Macro — batch runs

Define the run plan once (scheme set + Sat/Bri/Con). For each run, click the Areas cell to set how each area behaves for that run. One export bakes the full pack (character + every loaded mount). The plan persists; per-area treatments are per-class and reset when you load a new sprite.

Scheme sets

Each set is a scheme library (the same .json you export from the Schemes tab). Name it, then import. Runs reference sets by name.

Groups & masters

A group picks which scheme sets (colours) it covers and in what order. A master is a reusable Sat/Bri/Con + per-area lock preset, applied to one or more of the group's colours (the "Apply to" checkboxes). On export, every palette of one colour stays together: group → colour → master → scheme. Add more groups to stage blocks (e.g. basics before colourful) while keeping each block's order.
